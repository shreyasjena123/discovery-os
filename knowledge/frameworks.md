## JTBD (Jobs to Be Done)
Core question: What job is the user hiring this product to do?
When to apply: When a stated user request feels like a surface ask
Detection trigger: User describes a workaround, a ritual, or a 
frustration with existing tools
Key probe: "What were you doing before this? What does this 
let you stop doing?"

## Growth Loops
Core question: Which part of the growth loop does this touch?
Structure: Acquisition → Activation → Retention → Referral → Revenue
When to apply: When prioritizing features by business impact
Detection trigger: Feature request that affects how users find, 
adopt, or return to the product
Key probe: "Which loop node does this strengthen? Is that node 
our current constraint?"

## 7 Powers / Counter-Positioning
Core question: Why can't a competitor copy this immediately?
When to apply: When evaluating whether a feature is defensible
Detection trigger: Feature that incumbents could replicate easily
Key probe: "What would it cost an incumbent to copy this? 
What would they have to give up?"
