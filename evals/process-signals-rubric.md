# Process Signals Output Rubric

Scores any `process-signals` output on 4 criteria. Each criterion scores 0–2. Max score: 8.

---

## Criterion 1: Proxy Pain Detection

| Score | Description |
|-------|-------------|
| 2 | System identified a stated request and surfaced the deeper frustration behind it with specific evidence from the input |
| 1 | System flagged a possible proxy but evidence was weak |
| 0 | System took stated requests at face value |

---

## Criterion 2: Workaround Identification

| Score | Description |
|-------|-------------|
| 2 | System identified an existing workaround and assessed how entrenched it is |
| 1 | System mentioned a workaround but didn't assess entrenchment |
| 0 | System missed existing workarounds entirely |

---

## Criterion 3: Wrong Success Metric Flag

| Score | Description |
|-------|-------------|
| 2 | System identified what the user is protecting with current behavior that a solution might destroy |
| 1 | System noted friction but didn't identify what's being protected |
| 0 | System accepted surface metric as the real goal |

---

## Criterion 4: Research-Decision Bridge

| Score | Description |
|-------|-------------|
| 2 | System made a specific "what you should actually build" recommendation tied directly to evidence |
| 1 | System gave a vague directional recommendation |
| 0 | System summarized findings without a product recommendation |
